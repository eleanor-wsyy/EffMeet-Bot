# 会议干预机器人 v1.2 — 项目文档

---

## 1. 硬件配置

### 主控
- **ESP32-S3**

### 引脚定义

| 模块 | 引脚 | 变量名 | 说明 |
|------|------|--------|------|
| 5路循迹 — 最左 | IO4  | `PIN_FL`    | 巡线用 |
| 5路循迹 — 左中 | IO5  | `PIN_ML`    | 巡线用 |
| 5路循迹 — 中   | IO6  | `PIN_MID`   | 巡线用 |
| 5路循迹 — 右中 | IO7  | `PIN_MR`    | 巡线用 |
| 5路循迹 — 最右 | IO15 | `PIN_FR`    | 巡线用 |
| 计数循迹        | IO1  | `PIN_COUNT` | 安装于机器人左侧，转向/回程计数 |
| MG90S 舵机      | IO19 | `PIN_SERVO` | 执行器 |
| L298N — IN1     | IO21 | `IN1`       | 右电机方向 A |
| L298N — IN2     | IO47 | `IN2`       | 右电机方向 B |
| L298N — IN3     | IO14 | `IN3`       | 左电机方向 A |
| L298N — IN4     | IO13 | `IN4`       | 左电机方向 B |
| L298N — ENA     | IO40 | `ENA`       | 右电机 PWM（通道 0） |
| L298N — ENB     | IO41 | `ENB`       | 左电机 PWM（通道 1） |

> **传感器逻辑：黑线 = HIGH，白色 = LOW**

---

## 2. 可调参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `TRACK_SPEED`  | `230` | 循迹前进速度（PWM 0~255） |
| `ROTATE_SPEED` | `150` | 原地旋转速度（PWM 0~255） |
| `SERVO_ANGLE`  | `90`  | 舵机来回摆动角度（度） |
| `SERVO_STEP_MS`| `15`  | 舵机每步延时，越大越慢（毫秒） |
| `TURN180_TIME` | `900` | 掉头 180° 定时时长（毫秒） |

---

## 3. 方向编号规则

| 指令 | 方向 | 角度 |
|------|------|------|
| 1    | 正前 | 0°   |
| 2    | 右   | 90°  |
| 3    | 正后 | 180° |
| 4    | 左   | 270° |

- **初始朝向**：上电默认 `currentDir = 1`（0°）
- **旋转方向**：全部**顺时针**
- **转向步数公式**：`steps = (目标 - 当前 + 4) % 4`

---

## 4. MQTT 通信

| 参数 | 值 |
|------|----|
| 服务器 | `broker.emqx.io` |
| 端口 | `1883` |
| 订阅主题 | `esp32s3/control` |
| 发布主题 | `esp32s3/status` |
| 订阅消息格式 | 纯数字 `1` / `2` / `3` / `4` |
| 任务完成回复 | `done\|dir=<当前方向>` |
| 处理机制 | 任务执行期间（`busy=true`）忽略所有新指令，任务结束后才接收下一条 |

## 5. 完整动作流程

```
┌─────────────────────────────────────────────┐
│           收到 MQTT 指令（1/2/3/4）           │
│         busy=true，忽略后续新指令             │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│              计算转向步数                     │
│    steps = (目标 - 当前 + 4) % 4             │
│    steps=0 跳过旋转，直接对线                 │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│           原地顺时针旋转 steps × 90°          │
│   右轮前进 + 左轮后退，速度 ROTATE_SPEED      │
│   IO1 计数传感器：上升沿（白→黑）计一次        │
│   计数达到 steps → 停止旋转，延时 300ms        │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│            5路传感器微调对线（alignToLine）    │
│  MID=高 → 已对准，直接继续                   │
│  左侧看到线 → 微顺时针转，直到 MID=高         │
│  右侧看到线 → 微逆时针转，直到 MID=高         │
│  全白 → 慢速顺时针扫描，最多等 2000ms         │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│       循迹分级前进（trackToEnd）              │
│                                             │
│  传感器状态       动作                       │
│  ─────────────────────────────────────────  │
│  MID=黑         → 直行                      │
│  FL=黑          → 急左转（左停，右前进）      │
│  FR=黑          → 急右转（左前进，右停）      │
│  ML=黑          → 缓左转（左慢，右快）        │
│  MR=黑          → 缓右转（左快，右慢）        │
│                                             │
│  终点判断：FL+ML+MID+MR+FR 全为黑 → 停车     │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│           MG90S 舵机缓慢来回摆动              │
│                                             │
│  中位(90°) → 左端(90°-SERVO_ANGLE)          │
│           → 右端(90°+SERVO_ANGLE)           │
│           → 回中位(90°)                     │
│                                             │
│  每步 1°，延时 SERVO_STEP_MS ms（默认15ms）  │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│              原地差速掉头 180°                │
│   右轮前进 + 左轮后退，持续 TURN180_TIME ms   │
│   停止，延时 200ms                            │
│   再次调用 alignToLine 微调对线               │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│         循迹返回起点（trackBackToStart）      │
│                                             │
│  同样分级循迹逻辑保持在线上行进               │
│  同时持续检测 IO1（PIN_COUNT）               │
│  IO1 检测到黑线 → 再前进 300ms 精确停在中心  │
│  停车，延时 200ms                            │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│              更新方向 & 发布完成消息           │
│   currentDir = ((target-1+2) % 4) + 1       │
│   MQTT 发布：done|dir=<currentDir>           │
│   busy = false，恢复接收新指令               │
└─────────────────────────────────────────────┘
```

## 6. 关键函数说明

| 函数 | 说明 |
|------|------|
| `rotateSteps(steps)` | 原地顺时针旋转 steps×90°，IO1 上升沿计数 |
| `alignToLine()` | 旋转后用5路传感器微调，确保中线对准黑线 |
| `trackToEnd()` | 分级循迹前进，五路全黑判定终点 |
| `servoWave(angle)` | 舵机以 1° 步进缓慢来回摆动，SERVO_STEP_MS 控制速度 |
| `turn180()` | 定时差速掉头 180° |
| `trackBackToStart()` | 循迹返回，IO1 检测中心十字停车 |
| `doTask(target)` | 串联以上所有步骤的主任务函数 |
| `mqttCallback()` | 收到指令时，busy 状态下直接丢弃，否则写入 pendingCmd |

---

## 7. 状态变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `currentDir` | `int` | 当前朝向（1~4），上电初始为 1 |
| `busy` | `bool` | 任务执行中为 true，阻止接收新指令 |
| `pendingCmd` | `int` | 待执行指令缓存，-1 表示无指令 |
